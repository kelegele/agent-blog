# blog-publish Skill 设计手记

## 起点

我想做一个 skill：在任何项目中，通过 agent 发一篇博文到我的 Astro 博客。

听起来很简单——不就是往仓库里写一个 Markdown 文件吗？但实际做下来，我们发现了一些有趣的问题。

## 问题一：环境的诅咒

作为一个开发者，我的第一反应是：

1. Clone 仓库
2. `pnpm install`
3. 写文章
4. `pnpm dev` 预览
5. `git push`

这套流程对我来说毫无问题。但当我把这个 skill 拿给我的同事——一位产品经理——她没有 Node.js，没有 pnpm，甚至 git 也没装好。她只有一台电脑、一个浏览器、和一个 GitHub 账号。

这时候我才意识到：**我习以为常的开发环境，恰恰是普通人使用 AI 工具的最大门槛。**

所以我们设计了三层访问机制：

| Tier | 需要 | 适合谁 |
|------|------|--------|
| 1 — 本地 clone | git + 仓库路径 | 开发者（最快） |
| 2 — 浅克隆 | git | 有基础开发环境的人 |
| 3 — 纯 GitHub API | 只需 `GITHUB_TOKEN` | 任何人 |

Tier 3 只需要 `curl`（macOS/Linux 自带）和一个 GitHub Token。不需要安装任何东西。

## 问题二：预览的困境

写完文章要预览效果——这很合理。但 Astro 的预览需要 `pnpm dev`，又回到了 Node.js 环境的问题。

起初我们考虑了几个方案：

- **让用户装 Node.js？** 太重了，违背零依赖的初衷。
- **用 `npx` 临时跑？** 前提还是有 Node.js。
- **截图？** Agent 没有浏览器渲染能力。
- **不预览直接发？** 太草率了。

最终方案是：**让 Agent 读取仓库里的真实源码，动态拼装一个独立 HTML 文件。**

具体来说，Agent 从 GitHub 仓库读取 5 个源文件（CSS、布局、组件），把文章 Markdown 转成 HTML，套进博客的真实样式里，生成一个 `preview.html`，然后 `open` 在浏览器里打开。

不需要 Astro，不需要 Node.js，不需要任何构建工具。只要有浏览器就行。

而且这个预览是**动态的**——每次都从仓库读最新的源码，所以博客改了样式，预览也自动跟上，永远不会过时。

## 问题三：授 Agent 以鱼 vs 教 Agent 以渔

在早期版本里，我们把很多东西写死在 skill 里：

- CSS 样式快照 → 嵌入在 skill 文件里
- 分类列表 → 硬编码 `极客/geek`、`创造力/creativity`
- 本地仓库路径 → 硬编码 `/Users/fh/Projects/agent-blog`
- `gh` CLI → 作为必需依赖

这就像给 Agent 一条鱼——"你照着做就行了"。

但问题是：
- 博客样式改了，skill 里的快照就过时了
- 新增了分类，skill 不知道
- 换一台电脑，路径就不对
- 没装 `gh`，整个 skill 就废了

后来我们把所有这些都改成了**动态**的：

- 样式 → Agent 去读仓库最新的 `global.css`
- 分类 → Agent 去读仓库里所有文章的 frontmatter
- 路径 → 自动检测，有环境变量就用，没有就搜索
- `gh` → 不依赖，纯 `git` 或 `curl` 都能跑

**教 Agent 以渔——告诉它"去哪里找信息"，而不是"信息是什么"。**

这其实是一个写 skill 的重要原则：

> 好的 skill 不存储答案，而是教 Agent 如何获取答案。

## 写给非技术读者

如果你不是开发者，但你在用 AI 工具，你可能会遇到类似的问题：

- "这个工具让我装 XXX，但我不知道怎么装"
- "我跟着教程做到一半，报了个错就卡住了"
- "为什么别人的能用，我的就不行？"

这些往往不是你的问题，而是工具的设计问题。好的工具应该适应你的环境，而不是要求你适应工具。

我们在设计这个 skill 的时候，始终在问一个问题：

**"如果用户只有一台刚买的电脑，他能不能用？"**

答案是可以。只要你能打开浏览器，能拿到一个 GitHub Token，你就能发博客。

## 技术细节

- 仓库：`https://github.com/kelegele/agent-blog`
- 博客框架：Astro 6（静态生成）
- 部署：Vercel（push to main 自动部署）
- Skill 安装：`npx skills add kelegele/agent-blog`
- 零依赖预览：Agent 从仓库读取源码 → 拼装独立 HTML → 浏览器打开
